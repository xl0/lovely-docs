**Setup**: SvelteKit (`npx sv create`) or Vite recommended. **File structure**: `.svelte` files with optional `<script>`, `<script module>`, `<style>` sections; `.svelte.js/.ts` for reactive modules. **Runes** (`$state`, `$derived`, `$effect`, `$props`, `$bindable`, `$inspect`, `$host`): compiler keywords for reactivity. `$state` creates reactive proxies (objects/arrays deeply), `$state.raw` for non-mutating. `$derived` auto-updates on dependency changes, side-effect free. `$effect` runs on state changes with auto-tracking (sync only), return teardown function; avoid state updates. `$props` receives inputs with defaults/renaming, `$bindable` enables two-way binding. `$inspect` logs changes (dev-only). `$host` accesses custom element host.
